# PRD: 법령이지 (LawEasy) - 한국 법령 검색 공공서비스

**문서 버전:** v1.0
**작성일:** 2026-04-04
**상태:** Draft

---

## 1. 개요

### 1.1 프로젝트 요약

| 항목 | 내용 |
|------|------|
| **서비스명** | 법령이지 (LawEasy) |
| **콘셉트** | "누구나 쉽게 찾는 대한민국 법률" |
| **서비스 유형** | 공공서비스 웹 애플리케이션 |
| **대상 사용자** | 일반 국민, 공무원, 법률 실무자 |
| **GitHub 저장소** | `law-easy` |

### 1.2 배경 및 목적

현행 법제처 법령 검색 시스템은 전문가 중심의 인터페이스로, 일반 국민이 필요한 법률 정보를 쉽게 찾기 어려운 구조이다. 본 프로젝트는 법제처 Open API를 활용하여 **직관적이고 접근성 높은 법령 검색 서비스**를 제공하는 것을 목표로 한다.

### 1.3 핵심 가치

- **접근성**: 법률 비전문가도 쉽게 사용 가능
- **신뢰성**: 법제처 공식 데이터 기반
- **효율성**: 통합 검색으로 원스톱 법률 정보 제공
- **확장성**: 향후 AI 법률 질의응답 기능 추가 가능

## 2. 디자인 방향

### 2.1 디자인 콘셉트

**참고 템플릿:** Tribunal Authoritative International Landing Page Template (rocket.new)

공공기관의 **권위성과 신뢰감**을 유지하면서도 **현대적이고 직관적인** 사용자 경험을 제공한다.

### 2.2 디자인 원칙

| 원칙 | 적용 |
|------|------|
| **권위 있는 시각적 무게감** | 공공서비스로서의 신뢰감 전달 |
| **절제된 색상 팔레트** | 과하지 않은 공식 문서 느낌 |
| **명확한 타이포그래피 계층** | 정보 우선순위 직관적 전달 |
| **단일 열 흐름** | 위에서 아래로 자연스러운 탐색 |

### 2.3 색상 시스템

```
배경색:        #F5F0EB  (파치먼트 화이트 - 중성적, 무게감)
본문 텍스트:    #6B7B8D  (신중함 회색 - 절제된 톤)
제목/강조:     #3E2C1E  (챔버 오크 - 권위 있는 진한 색)
상호작용 요소:  #2B4C7E  (정의 파란색 - 버튼, 링크)
보조 강조:     #1A3A5C  (네이비 - 호버, 활성 상태)
경고/알림:     #C0392B  (레드 - 에러, 중요 알림)
성공:         #27AE60  (그린 - 성공 메시지)
```

### 2.4 타이포그래피

```
제목 (H1~H3):  Noto Serif KR (세리프 - 공식적, 권위)
본문/UI:       Pretendard (산세리프 - 현대적, 가독성)
코드/법조문:    D2Coding (모노스페이스 - 법조문 정렬)
```

### 2.5 핵심 UI 패턴

- **히어로 섹션**: 좌측 2/3 비주얼 + 우측 1/3 검색바
- **카드 기반 결과 목록**: 그림자와 여백으로 구분
- **아코디언 조문 뷰**: 펼치기/접기로 조문별 탐색
- **하이라이트 비교 뷰**: 신구대조 시 변경부분 색상 강조
- **반응형 디자인**: 데스크톱 우선, 모바일 단일 열 조정

## 3. 기능 요구사항

### 3.1 Phase 1 - 핵심 기능 (MVP)

#### 기능 ①: 법령 검색

| 항목 | 상세 |
|------|------|
| **설명** | 키워드 기반 법률·시행령·시행규칙 통합 검색 |
| **데이터** | 법제처 Open API |
| **우선순위** | P0 (필수) |

**세부 기능:**
- 키워드로 법령명, 조문 내용 통합 검색
- 약칭 자동 인식 (예: "화관법" → 화학물질관리법)
- 필터링: 법령 종류(법률/시행령/시행규칙), 소관부처, 시행일자
- 검색 결과 카드형 리스트 표시
- 법령 상세 페이지: 조문별 아코디언 펼치기
- 조문 직접 링크 공유 기능
- 검색어 하이라이트

**API 연동:**
- `search_korean_law`: 법령 검색
- `search_law_articles`: 조문 검색
- `read_legal_resource`: 법령 상세 조회

---

#### 기능 ②: 판례 검색

| 항목 | 상세 |
|------|------|
| **설명** | 키워드·사건번호 기반 판례 검색 |
| **데이터** | 법제처 Open API |
| **우선순위** | P0 (필수) |

**세부 기능:**
- 키워드, 사건번호로 판례 검색
- 법원 구분 필터: 대법원, 하급심, 헌법재판소
- 판례 요지 미리보기
- 판례 전문 상세 보기
- 관련 법령 자동 연결 링크
- 판례 날짜순/관련도순 정렬

**API 연동:**
- `search_korean_law`: 판례 검색 (type: precedent)
- `search_legal_terms`: 법률 용어 해석
- `search_statutory_interpretations`: 유권해석 검색

---

#### 기능 ③: 신구대조 비교

| 항목 | 상세 |
|------|------|
| **설명** | 법령 개정 전후 조문 비교 |
| **데이터** | 법제처 Open API |
| **우선순위** | P0 (필수) |

**세부 기능:**
- 특정 법령 선택 후 개정 이력 조회
- 개정 전(구법) / 개정 후(신법) 양쪽 분할 화면
- 변경된 부분 색상 하이라이트 (추가: 초록, 삭제: 빨강, 수정: 노랑)
- 개정 이력 타임라인 시각화
- 특정 개정 시점 선택 비교

**API 연동:**
- `search_korean_law`: 법령 검색
- `read_legal_resource`: 법령 상세 (시점별)
- `explore_legal_chain`: 법령 연혁 탐색

## 6. API 설계

### 6.1 내부 API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/search?q={keyword}&type={type}&page={page}` | 법령 검색 |
| GET | `/api/search/detail?id={lawId}` | 법령 상세 조회 |
| GET | `/api/precedent?q={keyword}&court={court}&page={page}` | 판례 검색 |
| GET | `/api/precedent/detail?id={caseId}` | 판례 상세 조회 |
| GET | `/api/compare?lawId={lawId}&date={date}` | 신구대조 비교 |
| GET | `/api/compare/history?lawId={lawId}` | 개정 이력 조회 |

### 6.2 외부 API (법제처 Open API)

- **인증**: OC 키 (환경변수 `LAW_API_KEY`)
- **형식**: XML → JSON 변환 처리
- **캐싱**: 검색 결과 5분, 법령 상세 1시간 캐시
